  │                                                             │
  └─────────────────────────────────────────────────────────────┘


2. THE BUSINESS PROBLEM WE SOLVED
══════════════════════════════════

BEFORE (manual, error-prone):
  • Product info scattered across 3 systems
  • Tax mismatches between POS and Shopify
  • Prices updated in one place, forgotten in another
  • No easy way to see "is this product set up everywhere?"
  • Ordering from vendors done on paper or memory
  • Shelf labels requested verbally, often forgotten

AFTER (unified, automated):
  • One screen shows product status across ALL systems
  • Tax mismatches detected instantly with red alerts
  • Split pricing is intentional and tracked
  • Real-time POS + Shopify lookup on every product
  • Digital vendor ordering with history
  • Label queue system — tap a button, label gets printed

4. KEY FEATURES — PRODUCT HUB
══════════════════════════════

The "Product Hub" is the heart of the app. When you open any
product, the system AUTOMATICALLY:

  ┌──────────────────────────────────────────────────────┐
  │  STEP 1: HYDRATION (happens in parallel, ~2 sec)    │
  │                                                      │
  │  ┌─────────────┐         ┌─────────────────────┐    │
  │  │ Look up SKU │         │ Look up SKU via     │    │
  │  │ in PLU.csv  │         │ Cloud Run Proxy →   │    │
  │  │ (POS data)  │         │ Shopify GraphQL API │    │
  │  └──────┬──────┘         └──────────┬──────────┘    │
  │         │                           │                │
  │         ▼                           ▼                │
  │  ┌──────────────────────────────────────────────┐    │
  │  │         CONFLICT DETECTION ENGINE            │    │
  │  │                                              │    │
  │  │  • Tax mismatch? → 🔴 CRITICAL ALERT        │    │
  │  │  • Price different? → 🟡 INFO (intentional)  │    │
  │  │  • Name mismatch? → 🟠 WARNING               │    │
  │  │  • Missing from system? → 🔴 NOT FOUND       │    │
  │  └──────────────────────────────────────────────┘    │
  │                                                      │
  │  STEP 2: DISPLAY unified product card                │
  │                                                      │
  │  ┌─ Status Bar ─────────────────────────────────┐    │
  │  │  ✅ POS ✓  ·  Shopify ✓           1 note 🔄 │    │
  │  └──────────────────────────────────────────────┘    │
  │  ┌─ Alert ──────────────────────────────────────┐    │
  │  │  🔴 TAX MISMATCH: POS=Non-Taxable,          │    │
  │  │     Shopify=Taxable                          │    │
  │  └──────────────────────────────────────────────┘    │
  │  ┌─ Product Image ─────────────────────────────┐    │
  │  │  [Shopify image]  apniroots.com IMAGE SOURCE │    │
  │  └──────────────────────────────────────────────┘    │
  │  ┌─ Split Pricing ─────────────────────────────┐    │
  │  │  STORE (→ POS):   $5.99  │ Case: $65.00    │    │
  │  │  ONLINE (→ Shop): $6.49  │ Case: $70.00    │    │
  │  │  COST:            $3.50  │ Case: $38.00    │    │
  │  └──────────────────────────────────────────────┘    │
  │  ┌─ Tax ────────────────────────────────────────┐    │
  │  │  POS Tax: Non-Taxable (code 4)  [MASTER]    │    │
  │  │  Shopify Tax: Taxable           [MISMATCH]  │    │
  │  └──────────────────────────────────────────────┘    │
  │  ┌─ Categories ─────────────────────────────────┐    │
  │  │  POS Dept: SNACKS (dropdown from PLU)        │    │
  │  │  Shopify Tags: Indian Spices, Snacks         │    │
  │  │  ☑ Category mapping confirmed                │    │
  │  └──────────────────────────────────────────────┘    │
  └──────────────────────────────────────────────────────┘


5. DATA AUTHORITY RULES
═══════════════════════

Each system is the "master" for specific data:

  ┌───────────────┬──────────────────────────────────┐
  │ DATA          │ MASTER SYSTEM                    │
  ├───────────────┼──────────────────────────────────┤
  │ TAX codes     │ 🏪 POS (Penny Lane)             │
  │               │    Code 1 = Taxable (HST 13%)   │
  │               │    Code 4 = Non-Taxable          │
  ├───────────────┼──────────────────────────────────┤
  │ IMAGES        │ 🛒 Shopify (apniroots.com)      │
  │               │    CDN-hosted product photos     │
  ├───────────────┼──────────────────────────────────┤
  │ PRICES        │ 📱 Firebase (our app)            │
  │               │    Store price → sent to POS     │
  │               │    Online price → sent to Shopify│
  │               │    Different prices = OK         │
  ├───────────────┼──────────────────────────────────┤
  │ CATEGORIES    │ ⚠️ Confirm-first (manual)        │
  │               │    POS has departments            │
  │               │    Shopify has tags               │
  │               │    User confirms mapping          │
  └───────────────┴──────────────────────────────────┘


6. COMPLETE FEATURE LIST
════════════════════════

  A. STORE MANAGEMENT
     • Multi-store support (Main Street, Oakville, Brampton)
     • Store-specific inventory and orders
     • Per-store label queue

  B. VENDOR MANAGEMENT
     • Vendor profiles with contact info
     • Products organized by vendor
     • Order history per vendor

  C. PRODUCT MANAGEMENT (Product Hub)
     • Unified view across POS + Shopify + Firebase
     • Auto-hydration: parallel POS & Shopify lookup
     • Conflict detection with severity levels
     • Split pricing: store vs online
     • Tax management: POS is master, Shopify mismatch alerts
     • Category mapping with confirmation workflow
     • Shopify image display (CDN source)
     • Front/back camera capture with background removal
     • Reorder rules (min stock, default order qty)

  D. ORDERING
     • Create orders from product list
     • Stock mode vs Order mode
     • Case/piece quantity tracking

  E. SYNC & EXPORT
     • Sync to Shopify (create or update products)
     • POS Export (generate NewCodes.txt for Penny Lane)
     • Bulk sync from hamburger menu

  F. LABEL QUEUE
     • One-tap label request from product screen
     • Reasons: Missing, Wrong Price, New Product, Damaged
     • Queue management screen for batch printing

  G. PRODUCT IMPORT
     • CSV import for bulk product loading
     • Sample imports included per store
